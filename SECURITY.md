# Security policy

## Supported versions

Security and disclosure fixes are applied to the latest tagged release and the
current `main` branch.

## Reporting a security or disclosure issue

Do not open a public issue for suspected credential exposure, sensitive-data
disclosure or a path that could reveal non-public infrastructure. Report those
cases by email to `lihongmin@edu.k.u-tokyo.ac.jp` with the subject line
`conditional-perturbation-benchmarking security report`.

For scientific correctness problems that contain no sensitive information,
open a GitHub issue and include the repository commit, command, platform and
minimal reproduction steps.

This repository intentionally contains no service credentials, cloud
locations, raw single-cell matrices or model checkpoints.
